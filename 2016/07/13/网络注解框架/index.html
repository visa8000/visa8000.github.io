Http网络请求

Veröffentlicht am 2016-07-13

背景

地图工程中，大部分业务方都是直接使用AsyncHttpClient发送网络请求，由于我们的请求中一般都需要一些基本参数和签名信息，这就造成各个业务方的请求中都需要自己写一套代码完成基本参数的拼装和签名，这就导致了大量冗余代码，另外部分业务方对参数encode并没采用引擎提供的方法，潜在的问题对一些特殊的符号可能会导致网络请求失败，鉴于上述问题，有必要对网络请求做进一步封装，业务方只需传入业务参数即可发起网络请求，不需要关心基本参数及签名。

业务方网络请求要素梳理

  1. 同步和异步:以同步或异步方式方式请求
  2. Get和Post： 以Get或Post方式发请求
  3. URL：请求的服务
  4. Param(get,post,optional)：get参数，post参数，参数可选
  5. CallBack：网络回调

接口封装（HttpRequestManager）

第一组：简单网络请求，传入参数发送请求，不自动添加签名和phoneinfo
/**
 * post请求
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数
 * @param bodyParams      post参数
 * @param fileParams      文件参数
 * @param responseHandler 回调
 */
public void postRequest(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                        HashMap<String, String> bodyParams,
                        HashMap<String, File> fileParams, AsyncHttpResponseHandler responseHandler) 
/**
 * get请求
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数
 * @param responseHandler
 */
public void getRequest(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                       AsyncHttpResponseHandler responseHandler)

第二组： 传入参数和token，自动添加phoneinfo和签名

/**
 * post请求，自动添加phoneinfo和签名，需要传入签名token
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数
 * @param bodyParams      post参数
 * @param fileParams      文件参数
 * @param responseHandler
 * @param token           签名token
 */
public void postRequest(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                        HashMap<String, String> bodyParams, HashMap<String, File> fileParams,
                        AsyncHttpResponseHandler responseHandler, String token)



/**
 * get请求
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数
 * @param responseHandler
 */
public void getRequest(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                       AsyncHttpResponseHandler responseHandler)

第三组： 发向PHPUI的请求，自动添加phoneinfo和签名

/**
 * 向PHPUI发送请求，自动添加phoneinfo和签名
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数，（参数中不能带phoneinfo参数）
 * @param bodyParams      post参数， （参数中不能带phoneinfo参数）
 * @param fileParams
 * @param responseHandler
 */
public void postRequestToPHPUI(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                               HashMap<String, String> bodyParams, HashMap<String, File> fileParams,
                               AsyncHttpResponseHandler responseHandler)
/**
 * 向PHPUI发送请求，自动添加phoneinfo和签名
 *
 * @param isSync          true表示同步，false表示异步
 * @param baseUrl
 * @param urlParams       get参数，（参数中不能带phoneinfo参数）
 * @param responseHandler
 */
public void getRequestToPHPUI(boolean isSync, String baseUrl, HashMap<String, String> urlParams,
                              AsyncHttpResponseHandler responseHandler)

注解含义介绍

@Post和@Get注解 方法注解

@Post(url = USER_LEVEL_HOST + USER_LEVEL_GET_PRIVACY_PATH, signToken = Constants.PHPUI_TOKE)

@Retention(RetentionPolicy.SOURCE)
@Target(ElementType.METHOD)
public @interface Post {
 /**
 * 请求的服务
 */
String url();

/**
 * 签名token，默认为空，为空时不添加phoneinfo和签名
 * token非空时，将自动添加phoneinfo信息和签名
 * token="PHPUI"时，表示是发向PHPUI的请求，自动添加phoneinfo信息和签名，签名采用的是引擎的方法
 */
String signToken() default "";
}

@Get 和post注解相似，它用来表示get请求

用post和get注解的方法，该方法参数类型支持String或基本类型，或AsyncHttpResponseHandler,post还支持File类型

@PostParam和@GetParam 参数注解

void test(@GetParam(value = “privacy_off”, optional = “if(isOpen)”) String off)

@Retention(RetentionPolicy.SOURCE)
@Target(ElementType.PARAMETER)
public @interface PostParam {
/**
 * 表示 注解参数的key
 */
String value();

/**
 * 该值用来表示 所注解参数是否是必选参数
 * 为空 时表示必传
 * 非空时 表示传该参数的条件，譬如 "if(param == 1)"
 */
String optional() default "";
}

@GetParam 同上

如果是必传参数，则不需给optional赋值，如果是可选参数，optional值的样式if(param == 1),即 当param =1时需要传入该参数

@GetMap和@PostMap参数注解

**
 * file: GetMap
 * 该注解可以用来标注 Get参数，参数类型必须是HasnMap<String,String>
 * 不建议使用此注解，无法在编译时检验一些特殊参数
 * <p/>
 * author: chenjie07
 * date: 2016/07/22 16:38
 */
@Target(ElementType.PARAMETER)
@Retention(RetentionPolicy.SOURCE)
public @interface GetMap {
}

@RegisterRequest 类注解

/**
 * 
 * 该注解用来标记一个接口类，该接口类中声明的方法需要用@Get或@Post进行注解
 * 用@RegisterRequest注解的接口，表示用来发起网络请求
 * 会自动生成接口类的实现类
 */
@Retention(RetentionPolicy.SOURCE)
@Target(ElementType.TYPE)
public @interface RegisterRequest {
/**
 * 超时时间，单位 ms
 *
 * @return
 */
int timeOut() default 10000;

String cookiePolicy() default "";
}

该注解用来标识一个接口类，该接口类中方法即是业务方网络请求接口，改接口类中的方法需要用@Post或@Get注解

注解使用约定：参数注解需要用在方法注解里面，方法注解需要包含在类注解里面
